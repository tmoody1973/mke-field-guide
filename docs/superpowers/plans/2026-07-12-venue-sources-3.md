## Decisions

1. **ComedySportz via SpotHopper JSON** (`https://www.spothopperapp.com/api/spots/8096/events`, public, CORS-open, no auth). Times: combine `event_date`'s DATE part + `start_time` (24h local) in America/Chicago — NEVER use `event_date`'s midnight-UTC time part. Venue from `linked.spots[0]` (ComedySportz Milwaukee, 420 South 1st Street, Milwaukee WI 53204). Events have NO per-event URL → event url = the listing page `https://cszmke.com/milwaukee-comedysportz-milwaukee-events` (repo convention for url-less items — check how existing parsers handle it; if a unique URL is required by persist, append `#ev{id}` fragment). Designed skip: `show_on_website !== true`.
2. **Improv via calendar crawl → detail JSON-LD.** Listing `https://improv.com/milwaukee/calendar/` (+ follow `a#moreshowsbtn[href^="?start="]` GETs, cap at `IMPROV_MAX_PAGES = 3` ≈ ~4-5 months); enumerate detail URLs from `.cal-list a.item[href]` (dedupe — multi-day runs repeat); fetch each detail page (cap `IMPROV_MAX_DETAIL_FETCHES = 40`/run, log drops) and parse ALL `Event` JSON-LD blocks (one per showtime; `startDate` ISO with offset — no year inference needed). Venue single: Milwaukee Improv, 20110 Lower Union Street, Brookfield WI 53045. Note `@type` is lowercase `"event"` in their JSON-LD — match case-insensitively. Multi-show days = separate records (same-show 15-min window keeps them distinct — different times).
3. **MSO via calendar grid + detail times.** Discovery: fetch bare `https://www.mso.org/concerts/calendar/` (follows current month), parse `#month_switcher option[value]` for the REAL month URLs (never construct month URLs — gap months are skipped by the site); fetch up to `MSO_MAX_MONTHS = 3` months ahead. Grid entries: `.calendar-event` per day-cell → `.show-title a[href]` (title + detail URL), day number from the enclosing `li.list-item-day` id/`.day-indicator`, year+month from the month URL path (bare-URL month = the `option[selected]` value). Times: `.event-time` LACKS am/pm → fetch each production's detail page (dedupe by URL; cap `MSO_MAX_DETAIL_FETCHES = 20`) and use `.performance-dates` lines ("7:30p on Saturday, September 26") as the authoritative time source, matched to the grid date; fall back to a pm-default heuristic (hours 1–7 → pm, 8–11 → am, 12 → pm) ONLY when the detail fetch fails, logged. Venue: parse the detail `.performance-range` tail ("at the Bradley Symphony Center, 212 W. Wisconsin Ave., Milwaukee"); **designed skip: non-metro localities** (e.g. Rhinelander) — skip with logged reason when the locality is absent from the metro allowlist (reuse/extend the existing metro/locality convention if one exists; else a small const list in the parser: Milwaukee, Wauwatosa, Brookfield, Bayside, Glendale, Franklin, Oak Creek, Shorewood, Whitefish Bay, West Allis).
4. **Rep via listing + detail HTML, date-range runs.** Listing `https://www.milwaukeerep.com/shows/current-season/`: `.show-listing[data-show-id][data-theater]` → title (`.show-listing-title`) + detail href. **LISTING DATE STRINGS ARE UNTRUSTED** (live-site bug: wrong end-years observed) — the detail page's `h2.tight-paragraph` range ("November 4 – December 14, 2025"; en dash; year at end only — apply it to both ends, rolling the start year back when start month > end month) is authoritative. Venue by `data-theater` map: 24 = Checota Powerhouse Theater, 25 = Stackner Cabaret, 30 = Herro-Franke Studio Theater (all at 108 East Wells Street, Milwaukee WI 53202), 26 = Pabst Theater (OFF-SITE — emit with the Pabst venue name/address so it consolidates onto the existing Pabst row); unknown id → fall back to detail-page venue text match, else skip logged. **Run shape: one event per production with day instances across the range** via the EXISTING day-range machinery (`day-range.ts` is frozen — consume it exactly as marcus/tribe day-instance expansion does, respecting its MAX_RANGE_DAYS clamp; a run longer than the clamp truncates with a logged, honest note). All-day (no times — no performance-level data is fetchable; dark-day inaccuracy [most theaters skip Mondays] is an accepted, documented limit — same class as other day-instance runs).
5. **Fetch budgets are per-run constants** (exported, tested where logic depends on them) — Improv 3 pages + 40 details, MSO 1 switcher + 3 months + 20 details, Rep 1 listing + ~12 details, CSZ 1 JSON. All within the ingest task's existing time budget; no schedule changes.
6. **Category tagging** rides the existing enrichment pipeline (no manual category writes in parsers beyond what the adapter interface already supports — match existing adapters).
